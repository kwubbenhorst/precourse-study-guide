HTML

  • HTML is one of three common languages used in web development along with CSS and JS.
  • HTML provides the structure of websites like the frame of a house.
  • The head element contains mostly metadata which apart from the title is for the browser not the user.
  • The body element represents the visible content shown to the user.
  • Elements with other elements nested inside are called parent elements whereas the nested elements are called child elements
  • Semantic elements which are elements whose name describes its role and content are preferred in the body over code that relates to format.
  • Tags are short words or abbreviations between the pointy brackets that open and close an element whereas the element is everything nested between in that block of code.
  • p is a commonly used tag to make a paragraph ul is for unnumbered list and li for list item
  • For screen readers it is important to use the headings h1 to h6 in descending order of size and to include along with an image source an alt text string describing the image. The alt description can also boost search engine hits

CSS

  • There are three ways to style a webpage using CSS: inline CSS, a way to change individual elements in the HTML code, internal CSS style sheet, a way to include style rules in an HTML head, and external CSS style sheet (most common), which keeps all style rules in a separate CSS file linked within the HTML head.
  • Separation of concerns is the concept that each section of code should have its own responsibility.
  • CSS rules are structured with a selector outside wavy brackets and a declaration block within wavy brackets and subdivided into properties and values separated by colons and separated by semi colons from the next property and value set. The selector defines what elements of the page the rules in the declaration block will apply.
  • Selectors can be two terms separated by a comma eg header and footer if you want the declaration block to apply to both. This helps to create don’t repeat yourself DRY code which is more compact and easier to maintain.
  • CSS box model means that each HTML element nests the content within layers of padding, border and margin. Margin refers to the space outside the element, padding to what is within the element. If you add a value to the padding you make the element bigger.
  • When CSS uses the wildcard or * as selector it means that all elements on the page will have the rule applied.
  • Besides text you can also apply CSS rules to images. The img selector will format all images in the markup. Display property and block value means every img starts on a new line. Imgs can also be formatted for height width and margin-left and margin-right.
  • Unnumbered lists ul can also be a selector for CSS rules formatted for padding-left and font-size Paragraphs p can also be a selector for CSS rules, formatted for text-align and font-size.
  • The CSS style sheet needs to be linked to the HTML code within the head in the index file after the last meta element and before the title element using link rel=“stylesheet” href=./assets/style.css” in pointy brackets.
  • Class selectors can be used in CSS to apply rules selectively to elements on a page. The period before the selector term designates it as a class selector. Eg .card makes a text box that can be valued in terms of various properties like width, margin, border, and box-shadow.
  • Colour values can be specified in English or using hexadecimal codes. Refer to the HTML color codes website for the code key.

Git

  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript

  • JavaScript is the tool that takes a static HTML/CSS webpage and makes it interactive (click buttons, retrieve data, form filling etc).
  • Basic JavaScript features include variables, strings, and data types.
  • Control flow is the order in which a computer executes code in a JavaScript file. It allows for the addition of conditional logic to applications.
  • Arrays are single variables that are used to hold a group of data (usually related data, maybe different data types). Forward loops operate on each element in an array which is useful when wanting to perform a task on multiple pieces of data at once.
  • JavaScript is an object-oriented programming (OOP) language. Concepts such as conditional logic and arrays will transfer to many other OOP languages.
  • Create an array by declaring a variable eg. var shapes = [“triangle”, “square”, “pentagon”, “circle”]; example of different data types held in an array: var studentInfo = [“Lu”, 54, true]; Note the string is in quotes but the number and boolean value are not
  • Individual data items stored in an array have a unique number called an index. This allows for working with each item in an array individually. The first item in any array has an index of 0, not 1, and index numbers for each item will go up by one whole number from there. Access to an individual item in an array is by this form: shapes[0]. It is the name of the variable that stores the array, then the index of the individual item you want in square brackets, in this case the first item in the sequence of the array.
  • A for loop enables a single code block to be executed on each item in an array. They are stopped by a conditional “if” statement making the execution operate a specific number of times rather than infinitely. In order to use the for loop, we need the shape's index and the array's length. We can find the length of an array by using a built-in property of an array, named “length.”
  • A for loop starts with “for” and contains 3 statements. The first, declaring a variable, determines the starting point of the loop var x = 0; second is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. Here the word “length” references the built-in array property: x
  • JavaScript itself is compact, but developers have written a variety of tools on top of the core JavaScript language, which include: Browser APIs (Application Programming Interfaces) which allow you to create HTML and CSS from within a browser, collect and manipulate webcam footage and generated 3D graphics and audio samples. Third party APIs which access eg. Twitter and Facebook, Third party frameworks and libraries which hasten the building of HTML applications. ChromeDevTools is one of the most useful browser APIs. It provides a web console for testing software in development. It is opened while viewing a VSCode file in the browser and entering the Mac command cmd+opt+I (as in igloo)
  • script files in JS need to be linked to the index file, so that the interactivity is applied to the HTML structure of the application. The link with the reference to the js file is added in the body part of the html code unlike the css file which was added to the head. It is done as the last line of code before the closure of the body.
  • Variables are containers that store values. They are what make applications dynamic. You start by declaring a variable with“let” followed by the variable name. Variable names are case sensitive and there is a list of invalid ones. A semicolon at the end of a line indicates where a statement ends.
  • Methods eg console.log (topics) are sets of instructions that can be executed by a computer. Here the word “console” refers to a test environment that developers use to check out their code. The “.log()” method will output whatever we add inside the parentheses to the console (here the variable, “topics”) which, being within parentheses functions as an “argument” in this statement.
  • The units of JS code are called “statements”.
  • After declaring a variable you give it a name eg. “MyVariable = Bob” or declare and give value on the same line “let MyVariable = ‘Bob.’” Later in this code this can be changed, and variables can be given values of different data types.
  • Different data types can be strings (text sequences, in single or double quotes), numbers (without quotes), boolean (true or false without quotes) arrays (multiple values stored in a single reference eg. Let MyVariable = [1, ‘Bob’, ‘Steve’, 10]; Members of the array can be referred to separately eg. MyVariable[0], MyVariable[1], etc.) and Objects (objects can be anything and can be stored in a variable).
  • Comments, rendered in this format /*Everything in between is a comment*/ or //This is a comment if there are no line breaks, are snippets of text that can be added along with code which the browser ignores.
  • An operator is a mathematical symbol that produces a result based on two values (or variables). Eg. Addition which adds two numbers or strings together, Subtraction, multiplication (uses an asterisk instead of an x) or division (uses a forward slash) which function as they would in math, assignment where you make a value equal to something Eg. Let MyVariable = ‘Bob,’ Strict equality === This performs a test to see if two values are equal and of the same data type. It returns a true/false (Boolean) result. Not, Does-not-equal, !, !==. This returns the logically opposite value of what it precedes. It turns a true into a false, etc.. When it is used alongside the Equality operator, the negation operator tests whether two values are not equal.
  • Conditionals are code structures used to test if an expression returns true or not. A very common form of conditionals is the “if…else” statement where the expression inside the “if () is the test. This uses the strict equality operator (as described above) to compare the variable ice cream with the string chocolate to see if the two are equal. If this comparison returns true, the first block of code runs. If the comparison is not true, the second block of code—after the else statement—runs instead.
  • Null is a special value in JavaScript that refers to the absence of a value.
  • Functions are a way of packaging functionality that you wish to reuse. You define a body of code as a function that executes when you call the function name in your code — a good alternative to repeatedly writing the same code. Functions look like variables but are followed by parentheses ().
  • Functions often take arguments, bits of data which go inside the parentheses, separated by commas if there is more than one argument.
  • The “return” statement tells the browser to return the result variable out of the function so it is available to use. This is necessary because variables defined inside functions are only available inside those functions. This is called “variable scoping.”
  • Event handlers, code structures that listen for activity in the browser and run code in response, are used for real interactivity on a website. Eg. The click event is handled by the browser.
  • Event handlers can be attached to elements in several ways.
  • JS is a dynamic language with dynamic types (ie. it is nominalistic) because variables are not directly associated with any particular value type and variables can be assigned values of all types
  • JS is also a weakly typed language because it allows implicit type conversion (ie. it practices auto correct) when an operation involves mismatched types instead of throwing type errors. For symbols and BigInts JS has intentionally disallowed certain kinds of implicit type conversion.
  • All types except object define immutable values represented directly at the lowest level of language called primitive values.
  • undefined indicates the absence of a value, while null indicates the absence of an object
  • The Boolean type represents a logical entity and is inhabited by two values: true and false.Boolean values are usually used for conditional operations.
  • The Number type has a range outside of which they will be represented by a double-precision floating point approximation.
  • The BigInt type is a numeric primitive in JavaScript that can represent integers with arbitrary magnitude. With BigInts, you can safely store and operate on large integers even beyond the safe integer limit for Numbers. A BigInt is created by appending n to the end of an integer or by calling the BigInt() function.
  • The String type represents textual data. Each element in the string occupies a position in the string. The first element is indexed at 0 with whole number increases of 1 along the string. JavaScript strings are immutable. This means that once a string is created, it is not possible to modify it. String methods create new strings based on the content of the current string, eg with substrings () and concatenations (+) or concat(). Use strings for textual data. When representing complex data, parse strings, and use the appropriate abstraction.
  • A Symbol is a unique and immutable primitive value. In some programming languages, Symbols are called "atoms". The purpose of symbols is to create unique property keys that are guaranteed not to clash with keys from other code.
  • An Object is a value in memory which is possibly referenced by an identifier. In JavaScript, objects are the only mutable values. Functions are, in fact, also objects with the additional capability of being callable.
  • Store arbitrary data in a Map rather than in objects.
  • Represent dates using the built-in date utility in JS.
  • Use arrays for ordered lists.
  • — Keyed collections are data structures take object references as keys. Set and Weak Set represent a collection of unique values, while Map and Weak Map represent a collection of key-value associations. Maps and WeakMaps make it easy to privately bind data to an object. WeakMap and WeakSet only allow garbage-collectable values as keys, thus are specifically used for memory usage optimization.
  • JSON (JavaScript Object Notation) is a lightweight data-interchange format, derived from JavaScript, but used by many programming languages. JSON builds universal data structures that can be transferred between different environments and even across languages.
  • JS has a standard library of built-in objects.
  • Type Coercion and Number coercion are employed by JS (autocorrect) but there are various rules that govern them making their corrections predictable. All data types (string, boolean, object etc) have their respective coercion processes except Null, Undefined and Symbol.
  • Control flow refers to the way JS code’s default linear (top to bottom) execution can be interrupted using conditional statements and loops
  • Conditional statements use “if” to introduce decision-making into the code. The form is “If (condition) { //block of code to be executed if condition is truthy}. If the condition is not truthy then the computer will ignore whatever is between the curly brackets and move on.
  • A value is considered false if one of the following values: 0, -0,On, “”, null, undefined, NaN, or Boolean data type with the value of false. All other values, including a Boolean with the value of true are considered truthy.
  • To create more than one condition use the form “else if (second condition)
  • Commenting out code in JS is done by inserting /* before the code block and */ after the code block. Commenting out means the code is inactive but not deleted in case it is needed later.
  • Functions are an important part of JavaScript and many other programming languages. A function is a set of instructions that tells the computer how to perform a certain task. They differ from for loops and conditional statements because they do not automatically execute when the js file is run, but have to be called by name. This is helpful because functions can be called again and again, and the programmer can control where in the control flow a task is executed or whether it is executed at all.
  • Create a function by using the function keyword followed by the unique name you want to give to the function. Eg. Function helloWorld() {//code block}. A function’s name is followed by parentheses ().
  • When wrapping for loops or if statements in functions, indent the for loop line
  • To call a function, use the name of the function in the JavaScript file where you want the code to execute.
  • When you want to randomly pick a different item from the array each time, you need to use a special Math function. Add a random topic variable that uses math.floor() and math.random() at the top of the js page (2nd line because the random topic variable will reference the variable topics which has not yet been defined, so it will need to be defined first in line 1). Math.floor() and math.random() are both properties of JavaScript's built-in Math object. Together they generate a random number.

  • Before the function selectTopic() is called/run, another explanatory statement can be inserted console.log(‘Which topic should we study first?’);
